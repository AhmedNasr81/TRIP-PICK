- **Color palette:** Keep the existing primary color but make sure it's used consistently. Use neutral grays for backgrounds, strong blacks for headings, muted tones for secondary text.

Animate the hero background with a slow subtle floating or parallax effect.
- **Sections:** Use proper section spacing (py-16 or py-24), section titles with a subtle accent line or badge above them.
- **Footer:** Structured multi-column footer with logo, links, and copyright.
- **Forms:** Clean inputs with proper focus rings, labels above inputs, good spacing.
- **Empty states:** Nice illustrated or icon-based empty states (no data found, no favorites, etc.)
- **Loading skeletons:** Replace any spinners in lists/grids with skeleton loaders that match the card shape.
- **Mobile:** Fully responsive — hamburger menu on mobile, stacked layouts, touch-friendly tap targets.

## Bug 1 — Removing a favorite from the Favorites page sends a POST instead of DELETE

In `src/hooks/api-hooks.ts`, the `useToggleFavorite` mutation has an `onSettled` callback that calls `queryClient.invalidateQueries({ queryKey: ['favorites'] })`. This re-fetches the favorites list from the server before the UI has updated, which causes the card to re-render with `is_favorited: true` and then fires a POST (add) instead of DELETE (remove).

**Fix:** Remove the favorites invalidation from `onSettled` — keep it only in `onError` for rollback. The optimistic update in `onMutate` already handles the UI.

Change `onSettled` from:
```ts
onSettled: (_, __, { id }) => {
  queryClient.invalidateQueries({ queryKey: ['favorites'] });
  queryClient.invalidateQueries({ queryKey: ['programs', id] });
},
```
To:
```ts
onSettled: (_, __, { id }) => {
  queryClient.invalidateQueries({ queryKey: ['programs', id] });
},
```

## Bug 2 — Email verification fires twice and breaks the token

In `src/pages/verify-email.tsx`, the `useEffect` with an empty dependency array runs twice in development due to React 18 Strict Mode intentionally mounting/unmounting components. This sends two requests to `GET /api/auth/verify-email?token=...`, and the second call fails or invalidates the token, causing the page to show an error even on a valid link.

**Fix:** Add a `useRef` guard so the verification only runs once.

Replace the current `useEffect` with:
```ts
const hasVerified = useRef(false);

useEffect(() => {
  if (hasVerified.current) return;
  hasVerified.current = true;

  const verify = async () => {
    const params = new URLSearchParams(window.location.search);
    const token = params.get("token");

    if (!token) {
      setStatus("error");
      setMessage("Invalid verification link. Token is missing.");
      return;
    }

    try {
      await api.get(`/api/auth/verify-email?token=${token}`);
      setStatus("success");
      setMessage("Your email has been successfully verified!");
    } catch (error: any) {
      setStatus("error");
      setMessage(error.response?.data?.detail || "Failed to verify email. The link may have expired.");
    }
  };

  verify();
}, []);
```
Also add `useRef` to the import at the top of the file.

## Bug 3 — Admin panel uses public endpoints instead of admin-specific ones

In `src/pages/admin/programs.tsx`, the page uses `usePrograms` (which calls `GET /api/programs`) instead of `GET /api/admin/programs`. The admin endpoint returns ALL programs including inactive ones, while the public endpoint only returns active programs — so inactive programs never show in the admin panel.

Same issue in `src/pages/admin/companies.tsx` — it uses `useCompanies` (public) instead of `GET /api/admin/companies`.

Also, the delete action in the admin programs page uses `useDeleteProgram` which calls `DELETE /api/programs/:id` (company-owner only), instead of `DELETE /api/admin/programs/:id` (can delete any program regardless of owner).

**Fix:**

1. Add these hooks to `src/hooks/api-hooks.ts`:
```ts
export const useAdminPrograms = (params?: Record<string, any>) => useQuery({
  queryKey: ['admin', 'programs', params],
  queryFn: async () => {
    const searchParams = new URLSearchParams();
    if (params) {
      Object.entries(params).forEach(([key, value]) => {
        if (value !== undefined && value !== null && value !== '' && value !== -1 && value !== '-1')
          searchParams.append(key, value.toString());
      });
    }
    return (await api.get<ProgramSimple[]>(`/api/admin/programs?${searchParams.toString()}`)).data;
  }
});

export const useAdminCompanies = (params?: Record<string, any>) => useQuery({
  queryKey: ['admin', 'companies', params],
  queryFn: async () => {
    const searchParams = new URLSearchParams();
    if (params) {
      Object.entries(params).forEach(([key, value]) => {
        if (value !== undefined && value !== null && value !== '' && value !== -1 && value !== '-1')
          searchParams.append(key, value.toString());
      });
    }
    return (await api.get<CompanyOut[]>(`/api/admin/companies?${searchParams.toString()}`)).data;
  }
});

export const useAdminDeleteProgram = () => {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: async (id: number | string) => (await api.delete(`/api/admin/programs/${id}`)).data,
    onSuccess: () => queryClient.invalidateQueries({ queryKey: ['admin', 'programs'] })
  });
};
```

2. In `src/pages/admin/programs.tsx`, replace `usePrograms` with `useAdminPrograms` and `useDeleteProgram` with `useAdminDeleteProgram`.

3. In `src/pages/admin/companies.tsx`, replace `useCompanies` with `useAdminCompanies`.
